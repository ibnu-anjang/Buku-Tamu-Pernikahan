M — Model

app/Models/GuestbookEntry.php 
class GuestbookEntry extends Model
{
    // Kolom yang boleh diisi (keamanan Mass Assignment)
    protected $fillable = [
        'name', 'phone', 'address', 'message'
    ];
}
"Hanya beberapa baris ini — Laravel sudah otomatis bisa menyimpan, membaca, mengupdate, dan menghapus data dari database MySQL. Tidak perlu tulis SQL manual."

C — Controller

app/Http/Controllers/GuestbookController.php 
// READ — Tampilkan halaman utama
public function index() {
    $entries = GuestbookEntry::latest()->get();  // ambil dari MODEL
    return view('welcome', compact('entries')); // kirim ke VIEW
}

// CREATE — Simpan ucapan baru
public function store(Request $request) {
    $validated = $request->validate([...]);   // validasi input
    GuestbookEntry::create($validated);        // simpan via MODEL
    return redirect()->back();
}
"Controller adalah penghubung — ambil data dari Model, kirim ke View. Validasi juga dilakukan di sini sebelum data disimpan."
Bagian 3 · MVC in Code

🗺️ Routes — Peta URL

routes/web.php 
// Halaman tamu
Route::get('/',          [GuestbookController::class, 'index']);
Route::post('/guestbook', [GuestbookController::class, 'store']);

// Admin CRUD
Route::prefix('admin')->group(function () {
    Route::get('/',           [...'adminIndex']); // Read
    Route::put('/{entry}',   [...'update']);     // Update
    Route::delete('/{entry}', [...'destroy']);   // Delete
});
"Routes adalah peta — URL mana ditangani oleh method mana di Controller. GET untuk membaca, POST untuk membuat, PUT untuk edit, DELETE untuk hapus."

Antisipasi Pertanyaan

Q & A

Q: Kenapa pakai Laravel, bukan PHP biasa?
Laravel menyediakan struktur MVC bawaan, validasi otomatis, ORM untuk database, dan banyak fitur yang membuat development lebih cepat dan kode lebih rapi.
Q: Apa bedanya GET, POST, PUT, DELETE?
GET = ambil/tampilkan data. POST = kirim data baru. PUT = update data. DELETE = hapus data. Ini mengikuti standar HTTP yang juga jadi dasar REST API.
Q: Kenapa ada $fillable di Model?
Untuk keamanan — membatasi kolom mana yang boleh diisi dari luar, mencegah hacker menyisipkan data ke kolom yang tidak seharusnya (Mass Assignment Protection).
Q: Bedanya welcome.blade.php dan admin.blade.php?
welcome.blade.php untuk tamu — hanya bisa mengisi form. admin.blade.php untuk pengelola — bisa melihat semua data dan melakukan CRUD penuh.
